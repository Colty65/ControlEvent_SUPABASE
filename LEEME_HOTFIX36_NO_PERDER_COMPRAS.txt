ControlEvent v15_prod - HOTFIX36 sin cambio de versión

Corrección sobre HOTFIX35.

Problema:
- Las donaciones ya se detectaban bien, pero las compras podían quedarse en 0.
- Esto podía ocurrir porque después de aplicar la tabla verdad del diagnóstico, alguna reconstrucción posterior dejaba solo las donaciones.

Cambios:

1. Blindaje final de compras
- Se añade ceHf36ForcePurchasesIfZero().
- Si después de todo el proceso la propuesta queda con 0 compras y hay prompt significativo, se generan compras mínimas de seguridad.
- Se aplica en tres sitios:
  a) tras recibir propuesta del backend,
  b) al pulsar "Usar diagnóstico como propuesta",
  c) dentro de renderProposal, antes de pintar y después de reconstruir.

2. Compras de seguridad
- Coca-Cola repartida: normal, zero y zero-zero.
- Fanta limón y naranja.
- Tónica si procede.
- Paella/comida: arroz, gambón, almejas.
- Barbacoa/cena: lomo, morcilla, panceta, pan.
- Aperitivo: patatas moderadas y berenjenas.
- Tardeo/cubatas: hielo, vasos y ginebra si procede.
- Reserva: Otras compras imprevistas 10%.

3. No se toca la lógica validada de donaciones
- Las 46 donaciones siguen saliendo de la tabla de diagnóstico.
- Este hotfix solo impide que la parte de compras quede vacía.

4. Cache bust
- planificacion-inicial.js?v=v15_prod_planificacion_hf36_no_perder_compras
- Marcador consola:
  HOTFIX36_NO_PERDER_COMPRAS_ACTIVO

No hay SQL nuevo.
No cambia versión visible.

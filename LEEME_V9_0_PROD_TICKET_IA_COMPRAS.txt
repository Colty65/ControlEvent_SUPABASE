ControlEvent v9.0_prod - Ticket IA para alta asistida de COMPRAS
=================================================================

Base: v8.5_prod + FIX48 validado.
Objetivo: automatizar la entrada de COMPRAS a partir de una foto de ticket, sin romper el flujo estable actual.

FUNCIONALIDAD NUEVA
-------------------
- Nueva opción inferior: "🧾IA", junto a los botones inferiores existentes.
- Visible solo para usuarios GD.
- Permite capturar foto con cámara o adjuntar imagen.
- Envía la foto a /api/receipt-ai/analyze para extraer líneas candidatas:
  producto, unidades, precio, importe, confianza.
- Presenta las líneas en pantalla para que el usuario pueda:
  - aceptar o desmarcar filas,
  - cambiar nombre de producto,
  - modificar unidades, precio o importe,
  - eliminar filas,
  - añadir filas manuales.
- Antes de grabar, el usuario elige:
  - TKxx,
  - tienda,
  - responsable.
- Al pulsar "Procesar y llevar a COMPRAS":
  - si el producto no existe, se crea en PRODUCTOS;
  - si existe, intenta actualizar precio por defecto y tienda por defecto;
  - si la protección de eventos Finalizados impide actualizar un producto usado en histórico cerrado, no se detiene la compra: se graba la compra con el precio del ticket y muestra aviso;
  - cada línea aceptada se graba en COMPRAS;
  - la foto se adjunta al TKxx seleccionado;
  - se recarga solo el evento seleccionado con FIX48.

IA / VARIABLES DE ENTORNO
-------------------------
La ruta de análisis usa OpenAI Responses API mediante fetch, sin añadir dependencias npm.

Variables necesarias en Vercel:
- OPENAI_API_KEY
  o bien
- CONTROLEVENT_OPENAI_API_KEY

Variable opcional:
- CONTROLEVENT_TICKET_AI_MODEL
  Por defecto: gpt-4.1-mini

Si no existe la API key, la pantalla permite seguir añadiendo filas manuales, pero el análisis IA devolverá aviso.

SEGURIDAD / NO ROMPER LO ESTABLE
--------------------------------
- No se toca BACKUP.
- No se toca CARGA/restauración.
- No se toca SQL.
- No se cambia FIX48.
- No se usa PUT /api/state.
- La escritura se hace por endpoints explícitos ya existentes:
  - /api/crud/productos
  - /api/crud/compras
  - /api/ticket-images
- Respeta evento Finalizado: no procesa tickets si el evento activo está Finalizado.
- La opción se oculta para RW y RO.

PRUEBA RECOMENDADA
------------------
1. Subir a GitHub/Vercel.
2. Añadir en Vercel OPENAI_API_KEY.
3. Refresco fuerte.
4. Entrar como GD.
5. Seleccionar un evento En curso.
6. Pulsar botón inferior "🧾IA".
7. Adjuntar una foto de ticket.
8. Analizar, revisar líneas, asignar TKxx, tienda y responsable.
9. Procesar.
10. Comprobar en COMPRAS que se han creado las líneas y que la foto queda en el TKxx.
11. Hacer backup y verificar que CE_COMPRAS_BBDD y CE_TICKET_IMAGES_BBDD aumentan coherentemente.

NOTA

Primera versión de funcionalidad. Está pensada como entrada asistida con revisión humana, no como carga automática ciega.

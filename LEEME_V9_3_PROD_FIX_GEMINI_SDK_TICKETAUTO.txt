ControlEvent v9.3_prod - FIX TicketAuto Gemini SDK + ajustes ventana IA

Cambios incluidos:

1) Gemini / IA
- Se sustituye la llamada REST manual por el SDK oficial @google/genai.
- Se añade dependencia @google/genai en package.json y package-lock.json.
- La inicialización se hace con new GoogleGenAI(), sin pasar la clave como parámetro.
- Antes de inicializar el SDK, ControlEvent normaliza variables para que funcione con:
  GEMINI_API_KEY, GOOGLE_API_KEY, CONTROLEVENT_GEMINI_API_KEY,
  GOOGLE_GENERATIVE_AI_API_KEY, OPENIA_API_KEY y, provisionalmente,
  OPENAI_API_KEY si no empieza por sk-.
- Modelo por defecto: gemini-2.5-flash.
- Cascada alternativa: gemini-2.5-flash-lite, gemini-2.0-flash, gemini-2.0-flash-lite.
- El error devuelto conserva proveedor/modelo para ver claramente si la llamada va por Gemini SDK.

2) Ventana TicketAuto
- Responsables: el desplegable muestra solo PERSONAS con rango SOCIO, ordenadas alfabéticamente.
- TKxx usado: si el ticket ya tiene líneas en COMPRAS para el evento activo, el desplegable queda en verde y al procesar avisa; el usuario puede cancelar o continuar.
- Fila manual: al elegir un producto existente se carga automáticamente su precio de PRODUCTOS; el precio queda editable.
- Líneas reconocidas: se conserva el orden visual del ticket/foto devuelto por la IA, para poder comprobar contra la imagen.
- Se fuerza nueva carga del JS con parámetro de caché actualizado.

Notas para Vercel:
- Revisar que existe GEMINI_API_KEY en Environment Variables.
- Tras cambiar variables de entorno, hacer Redeploy.
- El proyecto exige Node >=20 por @google/genai.

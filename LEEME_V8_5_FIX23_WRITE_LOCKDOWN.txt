ControlEvent v8.5_prod FIX23 - WRITE LOCKDOWN RAIZ

Objetivo:
Cortar de raíz cualquier escritura automática o masiva que pueda borrar o modificar datos sin una acción directa y controlada.

Reglas aplicadas:
1. Login, refrescar, cambiar evento, cambiar ventana, renderizar, abrir globos y ver fotos son SOLO LECTURA.
2. PUT /api/state queda bloqueado para guardados normales. Ya no sirve para mantenimiento de datos.
3. /api/crud-deltas queda eliminado: no se aceptan altas/bajas/modificaciones deducidas comparando estados del navegador.
4. /api/crud solo acepta escritura fila-a-fila si llega con cabecera nueva FIX23 y marca __crudRowOnly.
5. /api/ticket-images solo acepta subida/borrado si llega desde acción explícita de imagen con cabecera FIX23.
6. La subida de una foto ya NO crea eventos ni modifica eventos. Si el evento no existe, la foto se rechaza.
7. El service worker queda neutralizado y borra cachés antiguas para evitar scripts viejos.
8. Se ha parcheado también scripts/dev-server.js para que localhost no mantenga el comportamiento destructivo antiguo.

Consecuencia importante:
Esta versión es de CONTENCIÓN. Prioriza que no se pierda ni un registro más.
Algunas altas/modificaciones/bajas legacy que dependían de saveState / PUT /api/state pueden no persistir hasta reimplantar los mantenimientos con CRUD explícito botón por botón.

Estado correcto a futuro:
- Alta: POST /api/crud/<tabla> de una sola fila.
- Modificación: PUT /api/crud/<tabla>/<id> de una sola fila.
- Baja: DELETE /api/crud/<tabla>/<id> de una sola fila.
- Nunca guardar estado completo para mantenimiento ordinario.
- La restauración total de BACKUP es la única operación de reemplazo completo permitida.

ControlEvent v8.5_prod FIX31 - COMPRAS RPC DIRECTO

Objetivo:
- Mantener la BBDD congelada con global_write_lock=ON.
- Permitir únicamente COMPRAS mediante funciones SQL controladas.
- Cada botón de COMPRAS debe ejecutar UNA operación real:
  * Alta compra    -> ce_crud_compras_insert
  * Modificar      -> ce_crud_compras_update
  * Borrar         -> ce_crud_compras_delete
  * Situación evento -> ce_crud_eventos_situacion

PASO OBLIGATORIO antes de subir/probar esta versión:
1) Ejecutar en Supabase SQL Editor el fichero:
   ControlEvent_SQL_FIX31_COMPRAS_RPC_DIRECTO.sql

IMPORTANTE:
- No apagues global_write_lock.
- Las funciones SQL hacen bypass interno SOLO para su operación y validan evento En curso.
- Si el evento está Finalizado, COMPRAS no se inserta, no se modifica y no se borra.
- Si una baja no existe por id real, falla. No hay borrado por firma ni heurística.

Prueba recomendada con un evento EN CURSO de prueba:
1) Alta compra -> salir/entrar -> debe seguir.
2) Modificar compra -> salir/entrar -> debe quedar cambiada.
3) Eliminar compra -> salir/entrar -> no debe volver.
4) Finalizar evento -> intentar modificar compra -> debe bloquear BBDD.
